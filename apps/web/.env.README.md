# Environment Variables Reference

Quick reference for all environment variables used in Pocket Room.

## File Structure

```
apps/web/
├── .env.local.example    # Template (committed to git)
├── .env.local            # Your actual values (DO NOT COMMIT)
├── .env.production       # Production values (use in Vercel/deployment)
└── .env.test             # Test environment values
```

## Quick Setup

```bash
# 1. Copy template
cp .env.local.example .env.local

# 2. Generate encryption key
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# 3. Fill in your Supabase values from dashboard
# 4. Configure at least one auth provider
# 5. Restart server
npm run dev
```

## Variable Categories

### 🔐 Required (Must Configure)

| Variable | Description | Where to Get |
|----------|-------------|--------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL | Supabase Dashboard > Settings > API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public anonymous key | Supabase Dashboard > Settings > API |
| `SUPABASE_SERVICE_ROLE_KEY` | Admin key (server-only) | Supabase Dashboard > Settings > API |
| `TOKEN_ENCRYPTION_KEY` | 32-byte encryption key | Generate with crypto.randomBytes(32) |
| `NEXT_PUBLIC_APP_URL` | Your app URL | `http://localhost:3000` (dev) |

### 🚪 Gate Auth (User Login)

Configure in **Supabase Dashboard** > **Authentication** > **Providers**

| Variable | Description | Required |
|----------|-------------|----------|
| `NEXT_PUBLIC_GOOGLE_AUTH_CLIENT_ID` | Google OAuth client ID | Optional (for display) |
| `NEXT_PUBLIC_FEISHU_AUTH_APP_ID` | Feishu app ID | Optional (for display) |
| `NEXT_PUBLIC_WECHAT_AUTH_APP_ID` | WeChat app ID | Optional (for display) |

**Note:** Actual secrets are configured in Supabase Dashboard, not in `.env.local`

### 🤖 Provider Binding (AI Service Authorization)

| Variable | Description | Required |
|----------|-------------|----------|
| `OPENAI_CLIENT_ID` | OpenAI OAuth client ID | For OpenAI Companion |
| `OPENAI_CLIENT_SECRET` | OpenAI OAuth secret | For OpenAI Companion |
| `OPENAI_REDIRECT_URI` | OAuth callback URL | For OpenAI Companion |
| `GOOGLE_AI_CLIENT_ID` | Google AI client ID | For Gemini Companion |
| `GOOGLE_AI_CLIENT_SECRET` | Google AI secret | For Gemini Companion |
| `GOOGLE_AI_REDIRECT_URI` | OAuth callback URL | For Gemini Companion |
| `ANTHROPIC_CLIENT_ID` | Anthropic client ID | For Claude Companion |
| `ANTHROPIC_CLIENT_SECRET` | Anthropic secret | For Claude Companion |
| `ANTHROPIC_REDIRECT_URI` | OAuth callback URL | For Claude Companion |

### ⚙️ Optional Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `NODE_ENV` | `development` | Environment mode |
| `LOG_LEVEL` | `info` | Logging level (debug/info/warn/error) |
| `NEXT_PUBLIC_ENABLE_BROWSER_EXTENSION` | `true` | Enable extension features |
| `NEXT_PUBLIC_ENABLE_COMPANION` | `true` | Enable AI Companion features |
| `NEXT_PUBLIC_ENABLE_SEGMENT_SHARING` | `true` | Enable Segment sharing |
| `RATE_LIMIT_API` | `60` | API requests per minute |
| `RATE_LIMIT_AUTH` | `10` | Auth requests per minute |
| `RATE_LIMIT_AI_INVOCATION` | `20` | AI calls per minute |
| `NEXT_PUBLIC_SENTRY_DSN` | - | Sentry error tracking DSN |

## Security Checklist

- [ ] `.env.local` is in `.gitignore` (never commit)
- [ ] `TOKEN_ENCRYPTION_KEY` is unique and secure (32 bytes)
- [ ] `SUPABASE_SERVICE_ROLE_KEY` is never exposed to client
- [ ] Different keys for dev/staging/production
- [ ] OAuth redirect URIs match exactly (http vs https)
- [ ] Secrets are stored in deployment platform (Vercel Env Vars)

## Environment-Specific Values

### Development
```env
NEXT_PUBLIC_APP_URL=http://localhost:3000
NODE_ENV=development
LOG_LEVEL=debug
```

### Production
```env
NEXT_PUBLIC_APP_URL=https://your-domain.com
NODE_ENV=production
LOG_LEVEL=info
```

### Testing
```env
NEXT_PUBLIC_SUPABASE_URL=http://localhost:54321
NODE_ENV=test
LOG_LEVEL=error
```

## Troubleshooting

### Variable not loading?

1. Restart dev server after changing `.env.local`
2. Check variable name spelling (case-sensitive)
3. Ensure no spaces around `=` sign
4. Check file is named exactly `.env.local`

### OAuth not working?

1. Verify redirect URI matches exactly
2. Check client ID and secret are correct
3. Ensure provider is enabled in dashboard
4. Clear browser cookies and try again

### Encryption errors?

1. Verify key is exactly 32 bytes (base64 encoded)
2. Generate new key if corrupted
3. Note: Changing key invalidates all stored tokens
